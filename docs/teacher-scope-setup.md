# 先生アカウントの担当範囲設定

## 目的

公開URLで運用するとき、先生アカウントが全児童を見られる状態は便利ですが、教室・曜日・クラス単位で担当が分かれる場合は見える範囲を制限する必要があります。

この設定では、既存の先生アカウントはそのまま全児童を見られる状態に残し、必要な先生だけグループ指定で制限します。

## 事前条件

- `lesson_user_access` に管理者アカウントが登録済み
- `supabase/sql/admin_lesson_user_access_policies.sql` 実行済み
- 児童データの `group` に、現場で使うグループ名が入っている

## 初回に1回だけ実行するSQL

Supabase SQL Editorで次を実行します。

```sql
supabase/sql/teacher_group_scope_policies.sql
```

SQL Editorにはファイル名だけではなく、ファイルの中身を貼り付けて実行します。

## 管理者画面から設定する

1. 管理者アカウントでログインする。
2. 管理者画面の `Auth連携` を開く。
3. 先生の Auth User ID を入力する。
4. `先生の担当範囲` を選ぶ。
   - `全児童`: これまで通り全児童を表示
   - `グループ指定`: 指定したグループだけ表示
5. `グループ指定` の場合は、右の入力欄に `A組,B組` のようにカンマ区切りで入れる。
6. 先生登録を押す。

## 動作確認

先生アカウントでログインし、次を確認します。

- 指定したグループの児童だけが表示される。
- 指定外のグループの児童が表示されない。
- 先生アカウントでは児童の状態を確認できるが、練習結果・コイン・進捗は保存されない。
- 管理者アカウントでは全児童が表示される。

## 注意

- 児童の `group` と、先生の `scope_value` は完全一致で判定されます。
- `A組` と `Ａ組`、`月曜A` と `月曜Ａ` は別扱いです。
- 先生は閲覧専用です。児童データの追加・編集・削除・進捗保存は、管理者または生徒本人のアカウントで行います。
- 既存の先生を制限したい場合は、管理者画面で `グループ指定` に変更して再登録します。
- 既存の先生を再び全員表示に戻す場合は、管理者画面で `全児童` に変更して再登録します。
