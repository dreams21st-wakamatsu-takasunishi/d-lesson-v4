# 端末切替・同期確認手順

## 目的

児童が別PCで使うとき、または教室PCを入れ替えるときに、進捗・コイン・記録の保存漏れを防ぐ。

## 端末を変える前

1. 旧端末でDレッスンを開く。
2. 画面上部の保存状態を確認する。
   - `rls synced` または `synced`: 切替可能。
   - `syncing`: 数秒待ってから再確認。
   - `offline` / `sync error`: 切替前に通信とSupabase設定を確認。
3. 管理者画面 > バックアップ からJSONバックアップを保存する。
4. 必要に応じて、管理者画面 > 生徒の管理 > レポート で対象児童の状態を確認する。

## 新しい端末で行うこと

1. Dレッスンを開く。
2. Supabase Authでログインする。
3. 児童一覧に対象児童が表示されることを確認する。
4. 児童を選んで、コイン・進捗・記録が引き継がれていることを確認する。
5. 短い練習を1つクリアする。
6. 保存状態が `rls synced` または `synced` になることを確認する。

## 共有端末での注意

- 共有端末では、使い終わったらログアウトする。
- 別児童のログイン状態のまま使わない。
- `VITE_REQUIRE_SUPABASE_AUTH=true` の公開環境では、旧管理者パスワードに頼らない。
- 児童アカウントは、操作がないまま一定時間が経つと自動ログアウトする。初期値は20分。
- 自動ログアウトまでの時間を変える場合は、管理者画面 > 運用確認 > 共有端末の自動ログアウト で変更する。`VITE_STUDENT_IDLE_LOGOUT_MINUTES` は未設定時の初期値として使う。

## トラブル時

### 児童が表示されない

- `lesson_user_access` に対象Auth User IDと `user_data_id` が登録されているか確認する。
- 管理者画面 > Auth連携で登録済みアクセスを確認する。
- Supabase SQLでRLSポリシーが適用済みか確認する。

### 保存されない

- 保存状態が `offline` ならネットワークを確認する。
- `sync error` ならブラウザConsoleとSupabase RLSエラーを確認する。
- 復旧前に操作を続ける場合は、管理者画面からバックアップを保存しておく。

### 古い端末と新しい端末で内容が違う

- 両方の端末で保存状態を確認する。
- どちらのデータを正とするか決め、正しい端末でバックアップを取る。
- 必要な場合だけ、管理者画面の復元機能を使う。
