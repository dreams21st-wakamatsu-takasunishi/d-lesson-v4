# 現場運用開始までの残タスク

## 最優先

1. Auth / RLS の本番設定を固定する
   - 公開URLでは `VITE_REQUIRE_SUPABASE_AUTH=true`。
   - RLS検証完了後だけ `VITE_ENABLE_RLS_CLOUD_SYNC=true`。
   - `VITE_LEGACY_ADMIN_PASS` は公開URLに設定しない。
   - `VITE_ALLOW_LEGACY_ADMIN_PASS=false` を公開URLで固定する。

2. 児童データとAuthアカウントの紐づけ手順を固める
   - 新規児童は `student_...` の内部IDで作成される。
   - `lesson_user_access.user_data_id` には画面表示名ではなく、DB行の `id` を登録する。
   - 既存の名前ID行は、実名投入前に [児童IDの内部ID移行手順](internal-user-id-migration.md) に沿って整理する。
   - 管理者画面の「Auth連携」で、児童IDとAuth User IDを確認しながら登録する。
   - 初回管理者だけはSQL Editorで登録し、その後 `supabase/sql/admin_lesson_user_access_policies.sql` を実行する。
   - 先生・管理者・生徒の3ロールで、見えるデータと保存できるデータを確認する。

3. 実名投入前のバックアップ運用を決める
   - 初回運用前、学期末、児童追加・削除前に管理者画面からJSONバックアップを取る。
   - 復元前には現在データが自動バックアップとして保存される。
   - 復元テストを一度行い、どのファイルを正式バックアップとして保管するか決める。

## 運用前にほしい機能改善

1. Authユーザー紐づけの管理画面化
   - 管理者画面に「Auth連携」を追加済み。
   - SQL Editorで初回管理者を作った後、画面から「児童ID」「Auth User ID」「ロール」を登録・確認できる。
   - 現場テストでは、直接登録とSQLコピーの両方が使えることを確認する。

2. 先生アカウントの担当範囲
   - `lesson_user_access` に `scope_type` / `scope_value` を追加し、先生を全児童またはグループ指定で登録できるようにした。
   - 管理者画面の「Auth連携」で、先生の担当範囲を `全児童` / `グループ指定` から選べる。
   - 初回は `supabase/sql/teacher_group_scope_policies.sql` をSQL Editorで実行する。

3. 管理者操作の監査ログ
   - 管理者画面のバックアップ欄に、管理者操作ログを追加済み。
   - 児童追加、削除、表示名変更、生年月日変更、進捗編集、復元などを履歴に残す。
   - ログは児童データへ同期せず、管理端末のみに保存する。

4. 復元の安全機能
   - 復元前に現在データを自動バックアップする機能を追加済み。
   - 復元ファイルの日時、児童件数、設定データ有無を確認してから実行する。
   - RLS運用では、児童削除と復元時の古い行削除のため `supabase/sql/admin_user_data_delete_policies.sql` を実行しておく。

## あると現場が楽になる機能

1. 児童検索・絞り込み
   - 管理者画面の児童一覧に、名前/ID/グループ検索、学年絞り込み、グループ絞り込みを追加済み。

2. 一括編集
   - 管理者画面から児童CSVを出力し、表示名・生年月日・グループを一括更新できる機能を追加済み。
   - 更新は `user_data_id` を基準に行うため、同姓同名でも取り違えにくい。

3. 進捗レポート
   - 管理者画面で児童を選択し、個別レポートを確認・印刷できる機能を追加済み。
   - マウス、キーボード、ビジョン、ことば入力、文章練習、苦手キーをまとめて確認できる。

4. 端末切り替えの案内
   - 管理者画面に「運用確認」を追加済み。
   - 同じ児童が別端末で使うときのログイン手順、同期の確認方法、オフライン時の扱いは [端末切替・同期確認手順](device-handoff-guide.md) にまとめた。

## 後で整理したい設計

1. `__GLOBAL_SETTINGS__` の分離
   - `lesson_settings` テーブルへの任意移行を追加済み。
   - 初回は `supabase/sql/lesson_settings_table.sql` を実行し、`VITE_ENABLE_SETTINGS_TABLE=true` にする。
   - 診断は `supabase/sql/verify_lesson_settings.sql` で行う。
   - 詳細は [lesson_settings 移行手順](lesson-settings-table.md) にまとめた。

2. 旧パスワード導線の完全撤去
   - 現在は本番ビルドとAuth必須環境で旧パスワードが無効になるようにした。
   - Authロール運用が安定したら、旧パスワード入力UI自体を削除する。

3. `main.js` の分割継続
   - 大きいファイルにゲーム・UI・報酬・CSV出力がまだ混ざっている。
   - 今後の未定義エラーを減らすには、機能単位で `src/ui` / `src/games` へ移す。

## 現場テストの合格ライン

- 管理者が児童追加、表示名変更、生年月日変更、進捗編集、バックアップ、復元を行える。
- 先生が担当グループの児童だけを確認できる。全児童担当にした先生は全児童を確認できる。
- 先生アカウントでは児童データを書き換えず、確認操作後も再読み込み後もコイン・進捗が変わらない。
- 生徒が自分のデータだけを読み書きできる。
- 未ログイン、未登録Authユーザー、別生徒アカウントで児童一覧や進捗が見えない。
- ステージクリア後に、コイン・進捗・おすすめ・プログレスバーが更新される。
- 1日以上運用して、同期エラーや保存漏れがない。
