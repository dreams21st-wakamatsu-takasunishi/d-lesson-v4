# 管理者操作手引書

## 管理者アカウントの役割

管理者は、児童データ、Auth連携、バックアップ、復元、設定変更を扱います。  
本番環境では、旧式の管理者パスワードではなく、Supabase Auth の管理者ロールで操作します。

## 管理者ログイン

1. Dレッスンを開く。
2. 先生・管理者ログインを開く。
3. 管理者メールアドレスとパスワードでログインする。
4. タイトル画面に `管理者用` が表示されることを確認する。
5. 管理者画面を開く。

## 児童管理

管理者画面で行える主な操作:

- 児童追加
- 表示名変更
- 生年月日変更
- グループ変更
- 児童削除
- コイン付与
- 進捗編集
- レポート確認
- 取り組み履歴確認

新規児童は、内部ID `student_...` で保存されます。  
画面に表示する名前は `data.displayName` として保存します。

## 児童追加

1. 管理者画面を開く。
2. 児童管理の追加欄へ入力する。
3. 表示名、生年月日、グループを確認する。
4. 追加する。
5. 児童一覧に表示されることを確認する。
6. 必要に応じてAuth連携を行う。

## 児童削除

削除前に必ずバックアップを保存します。

1. 管理者画面 > バックアップでJSONバックアップを保存する。
2. 削除対象の児童を選択する。
3. 削除する。
4. 画面を再読み込みして復活しないことを確認する。
5. Supabase側にも古い行が残らないことを確認する。

## 進捗編集

進捗編集では次を変更できます。

- マウスLv
- キーボード進捗
- 所持アイテム

注意:

- 所持アイテムから現在使用中のテーマやエフェクトを外すと、自動的に `default` に戻ります。
- 実運用中は、理由がある場合だけ編集します。
- 編集後は児童本人で再読み込みし、表示が反映されているか確認します。

## 児童レポート

児童レポートでは次を確認できます。

- 全体進捗
- 現在の状態
- 苦手キー
- 文章練習
- ビジョントレーニング
- 直近の取り組み

必要に応じて印刷できます。

## 取り組み履歴

管理者画面の取り組み履歴では、日付ごとに児童の練習状況を確認できます。

確認できるもの:

- 日付
- 時刻
- 児童名
- 学年
- グループ
- 練習名
- 内容
- コイン

CSV出力もできます。

## Auth連携

Auth連携は、Supabase AuthのユーザーとDレッスン内の児童データを紐づける操作です。

扱うもの:

- Auth User ID
- user_data_id
- role
- 先生の担当範囲

role:

- `student`: 児童本人
- `teacher`: 先生
- `admin`: 管理者

注意:

- 児童のAuth User IDは1人ずつ異なります。
- 児童にはAuth User IDを伝えません。
- 先生の担当範囲をグループ指定にする場合、児童データの `group` と完全一致させます。

## バックアップ保存

次のタイミングで保存します。

- 本番運用開始前。
- 児童を追加・削除する前。
- バックアップ復元前。
- 一括変更前。
- 学期末。

バックアップファイルには児童情報が含まれるため、取り扱いに注意します。

## バックアップ復元

復元は強い操作です。必ず次を確認してから行います。

1. 現在データのバックアップを保存する。
2. 復元するファイルの日付を確認する。
3. 復元するファイルの児童数を確認する。
4. 復元する。
5. 画面を再読み込みする。
6. 児童追加・削除・進捗・操作ログが想定通りか確認する。

## 管理者操作ログ

管理者操作ログは、管理端末側に記録されます。

記録対象:

- 児童追加
- 児童削除
- 表示名変更
- 生年月日変更
- グループ変更
- コイン付与
- 進捗編集
- バックアップ
- 復元
- Auth連携
