# アカウント・セキュリティ手引書

## 基本方針

Dレッスンの本番運用では、情報漏洩防止を最優先にします。

守る情報:

- 児童名
- 生年月日
- グループ
- 進捗
- コイン
- 取り組み履歴
- ログイン情報

## 公開URLで守ること

- ログイン前に児童名一覧を表示しない。
- 旧式の管理者パスワードを公開URLに置かない。
- Supabase Authでログインを必須にする。
- RLSで読めるデータを制限する。
- 児童は本人データだけ読み書きできる。
- 先生は担当範囲だけ読める。
- 管理者だけが管理操作を行える。

## Supabase Auth User IDとは

Supabase Auth User IDは、Supabaseがログインアカウントごとに持つUUIDです。  
Dレッスンでは、このIDと `user_data.id` を `lesson_user_access` で紐づけます。

児童1人につき、原則として1つのAuth User IDが必要です。  
1つの児童アカウントを複数児童で共有すると、情報漏洩やなりすましのリスクが上がります。

## 児童ログイン

児童は次の2つでログインします。

- 児童番号
- あいことば

内部的には、児童番号からログイン用メールアドレスを組み立て、Supabase Authでログインします。  
児童にはメールアドレスやAuth User IDを伝えません。

## 児童ログインカード

カードに載せるもの:

- DレッスンURL
- 児童番号
- あいことば

カードに載せないもの:

- Auth User ID
- SupabaseのURL
- 内部メールアドレス
- 管理者情報

紛失時:

1. 対象児童のSupabase Authパスワードを変更する。
2. 管理者画面の `生徒管理` で対象児童を選択し、カードを再発行する。
3. 古いカードは使わないようにする。

詳しい印刷手順は [児童ログインカード印刷手順](../student-login-card-printing.md) を参照してください。

## 先生アカウント

先生アカウントは、確認用です。

- 全児童を見られる先生。
- 指定グループだけ見られる先生。

どちらにするかは管理者が決めます。  
担当外の児童が見える場合は、`lesson_user_access.scope_type` と `scope_value` を確認します。

## 管理者アカウント

管理者アカウントは少数に限定します。  
管理者は児童データの追加・編集・削除・復元ができるため、パスワードを共有しないでください。

推奨:

- 管理者アカウントは必要最小限。
- パスワードは強いものにする。
- 退職・担当変更時はパスワード変更またはアカウント停止。
- 公開URLに旧管理者パスワードを設定しない。

## RLSの役割

RLSは、Supabase側でデータの読み書きを制限する仕組みです。  
フロントエンドの表示制御だけではなく、データベース側でも制限します。

想定:

- 未ログイン: 児童データを読めない。
- 児童: 自分の `user_data` だけ読める、保存できる。
- 先生: 担当範囲の児童だけ読める。児童データは保存変更しない。
- 管理者: 管理に必要な範囲を読める、更新できる。

## 本番環境で設定しないもの

次は公開URL用の環境変数に設定しないでください。

```text
VITE_LEGACY_ADMIN_PASS
VITE_ENABLE_LEGACY_SUPABASE_SYNC=true
VITE_ALLOW_LEGACY_ADMIN_PASS=true
```

## 情報漏洩が起きる可能性のある場面

- ログイン前に児童名一覧を表示してしまう。
- 児童番号とあいことばを他人に見せる。
- 共有端末でログアウトしない。
- 先生アカウントの担当範囲が広すぎる。
- 管理者アカウントを共有する。
- バックアップファイルを安全でない場所に置く。
- Gitに児童情報入りCSVやバックアップを入れる。
- Supabase RLSが無効、またはpublic policyが残っている。

## 防止策

- 児童カードは本人だけに配る。
- 共有端末ではログアウトを徹底する。
- 自動ログアウト時間を現場に合わせる。
- バックアップファイルは管理者のみ扱う。
- `migration/student-login-accounts.csv` などの認証情報入りファイルはGitに入れない。
- 公開前に `npm.cmd run check:release` と `npm.cmd run check:public-latest` を実行する。
- Supabase SQLでRLS検証を行う。
