# 日常運用手引書

## 目的

教室でDレッスンを毎日使うための基本手順です。  
児童が練習し、先生が確認し、管理者が必要な管理操作を行う流れをまとめます。

## 使うURL

本番公開URL:

```text
https://dreams21st-wakamatsu-takasunishi.github.io/d-lesson-v4/
```

ローカル確認URL:

```text
http://127.0.0.1:5174/
```

ローカル起動:

```powershell
cd "C:\Users\conta\Desktop\pcれんしゅう開発\Dレッスン5.0\d-lesson-v4"
npm.cmd run dev:local
```

## 1日の基本の流れ

1. 端末でDレッスンのURLを開く。
2. 児童は自分の児童番号とあいことばでログインする。
3. 練習メニューから練習を選ぶ。
4. 練習終了後、コイン・進捗・取り組み履歴が保存される。
5. 終了時はログアウトする。
6. 必要に応じて先生または管理者が確認する。

## 授業前の確認

- 公開URLまたはローカルURLが開く。
- ログイン画面が表示される。
- ログイン前に児童名一覧が表示されない。
- 児童番号とあいことばでログインできる。
- Consoleにアプリ本体由来のエラーがない。

`content.js` や `message port closed` の単独エラーは、ブラウザ拡張機能由来の可能性が高いです。  
ただし、`ReferenceError`、`is not defined`、`Cannot read properties of undefined`、Supabaseの認証エラーは確認対象です。

## 授業中の注意

- 児童は自分の番号だけでログインする。
- 他の児童の番号やあいことばを使わない。
- 共有端末では、前の児童がログインしたままになっていないか確認する。
- 操作が終わったらログアウトする。
- 通信が不安定なときは、同じ操作を何度も繰り返す前に先生へ伝える。

## 授業後の確認

管理者または先生が必要に応じて確認します。

- 児童の進捗が更新されている。
- コインが増えている。
- 取り組み履歴に、練習内容と時刻が残っている。
- つぎのおすすめが表示される。
- 別端末で開いても進捗が引き継がれる。

## バックアップの目安

管理者画面からJSONバックアップを保存します。

- 本番運用開始前。
- 児童をまとめて追加・削除する前。
- バックアップ復元を行う前。
- 学期末。
- 大きな設定変更の前。

復元前には現在データの自動バックアップもありますが、重要な作業前は手動バックアップも保存してください。

## 先生確認モード

先生アカウントでは、練習画面の確認はできますが、児童データを書き換えない設計です。

確認方法:

1. 先生アカウントでログインする。
2. 対象児童を開く。
3. ステージを確認する。
4. 操作直後のコインや進捗を見る。
5. 画面を再読み込みする。
6. コインや進捗が変わっていないことを確認する。

先生確認モードで児童データが保存変更される場合は、運用を止めて修正対象にしてください。
