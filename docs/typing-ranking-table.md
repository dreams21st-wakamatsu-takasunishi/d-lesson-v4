# タイピングゲーム匿名ランキング設定

タイピングゲームのランキングを公開URL運用で使う場合は、児童データ本体ではなく `lesson_typing_rankings` に匿名スコアだけを保存します。

## 目的

- 児童名をランキング用テーブルに保存しない
- 児童は自分のスコアだけを書き込める
- 全ログインユーザーは匿名ラベルとスコアだけを読める
- テーブル未作成でも、Dレッスン本体の練習は止まらない

## 初回設定

Supabase SQL Editorで次のファイルの中身を実行します。

```text
supabase/sql/lesson_typing_rankings.sql
```

続けて、反映漏れがないか次の確認SQLを実行します。

```text
supabase/sql/verify_lesson_typing_rankings.sql
```

合格条件:

- ポリシー一覧に `SELECT` / `INSERT` / `UPDATE` の3つのポリシーが表示される
- 最後の結果テーブルで `result` がすべて `OK`

Supabase SQL Editorで結果表が1つだけ目立って見える場合は、最後に表示される `check_name` / `result` / `detail` の表を確認します。

実行後、タイピングゲームを1回プレイすると、結果画面のランキングが「クラス匿名ランキング」に切り替わります。

確認時は、児童アカウントでタイピングゲームを1回クリアし、再読み込み後もランキングに匿名ラベルとスコアが残ることを見ます。

## 表示仕様

- 自分の行は `自分` と表示します。
- 他児童は `児童 001` のような匿名ラベルで表示します。
- 実名、学年、グループ、生年月日などはランキングテーブルに保存しません。
- タイピングゲームメニューの「ランキングを見る」から、再読み込み後もクラウド上に記録が残っているか確認できます。
- 上位5位に入った児童だけ、ニックネームを登録できます。
- ニックネームには実名、名字、学校名、人を傷つける言葉、電話番号、メールアドレス、URLのような文字列を使えません。

## 注意

外部サイトの寿司打・Popタイピングのスコアは自動取得しません。Dレッスン側では、外部サイトを開いた時刻と閉じた時刻だけを取り組み記録に保存します。
